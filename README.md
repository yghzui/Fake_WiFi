# ESP32-CAM 模拟 Wi-Fi (打卡/中继) 使用步骤文档

本文档将指导您如何配置和使用基于 ESP32-CAM 的模拟 Wi-Fi 工具。该工具可以根据您的需求自定义 Wi-Fi 名称 (SSID)、密码、信道以及 BSSID（MAC 地址），常用于模拟特定的办公 Wi-Fi 环境以实现打卡功能。

此外，本工具现已支持**中继模式（Bridge Mode）**，在发出自定义 Wi-Fi 的同时可连接到上游网络，如果开发环境支持（如 `lwip_napt`），还可为连接的设备提供互联网访问。

---

## 1. 硬件要求

* **ESP32-CAM 开发板**（或其他 ESP32 开发板）
* **下载器/底板**（通常使用带有 **CH340** USB 转串口芯片的底板连接到电脑）
* 数据线（连接电脑和开发板）

---

## 2. 准备工作：检查与安装 CH340 驱动

由于开发板使用 CH340 芯片与电脑通信，您的电脑需要安装相应的驱动才能被识别。

1. **自动安装（推荐）**：
   - 找到项目目录中的 `install_ch340.ps1` 脚本文件。
   - 右键点击该文件，选择 **“使用 PowerShell 运行”**。
   - 脚本会自动检测您的系统是否安装了 CH340 驱动：
     - 如果已安装，脚本会提示无需安装。
     - 如果未安装，脚本会自动从官方下载并弹窗请求安装（请在弹出的管理员权限提示中点击“是”）。
2. **手动安装**（如果自动安装失败）：
   - 请访问官方下载页面：[CH341SER.EXE](https://www.wch.cn/downloads/CH341SER_EXE.html) 下载安装即可。
3. 安装完成后，将 ESP32-CAM 插入电脑，打开“设备管理器”，在“端口 (COM 和 LPT)”下应能看到 `USB-SERIAL CH340 (COMx)`，记住这个 COM 端口号。

---

## 3. 烧录代码

1. 确保您的电脑已安装 **Arduino IDE**。
2. 打开 Arduino IDE，进入 `文件` -> `首选项`，在“附加开发板管理器网址”中添加：`https://dl.espressif.com/dl/package_esp32_index.json`
3. 进入 `工具` -> `开发板` -> `开发板管理器`，搜索 `esp32` 并安装。
4. 使用 Arduino IDE 打开本项目的 `Fake_WiFi/Fake_WiFi.ino` 文件。
5. 在 `工具` 菜单中进行如下配置：
   - **开发板**: 选择 `AI Thinker ESP32-CAM` (或其他对应的 ESP32 型号)
   - **端口**: 选择刚才在设备管理器中看到的 COM 端口
6. 点击 **“上传”** 按钮。等待下方提示“上传成功”。
   > **注意**：部分 ESP32-CAM 在上传时需要按住开发板上的 `BOOT` 按键，或短接 `IO0` 和 `GND`，上传完成后断开并按一下 `RST` 按钮重启。如果是自带一键下载功能的底板，则不需要此操作。

## 4. 网页配置与使用（新版）

代码烧录成功后，设备默认会发出一个供您进行配置的热点。

### 步骤 1：连接默认配置网络
1. 打开手机或电脑的 Wi-Fi 列表。
2. 搜索并连接到名为 **`Fake_WiFi_Setup`** 的 Wi-Fi 网络（默认无密码）。

### 步骤 2：登录 Web 管理后台（Basic Auth）
1. 连接成功后，打开浏览器。
2. 在地址栏输入：**`http://192.168.4.1`**。
3. 浏览器会弹出账号密码输入框（HTTP Basic Auth）。
4. 当前默认账号密码（代码默认值）：
   - 用户名：`ad321`
   - 密码：`ad123`
5. 登录后会进入“配置查看页（/status）”。

> 安全建议：请在代码中修改默认账号密码，位置为 `Fake_WiFi/Fake_WiFi.ino` 的 `AUTH_USER` 和 `AUTH_PASS`。

### 步骤 3：认识页面结构
新版后台分为多个页面：

- **查看页 `/status`**：查看当前生效配置、管理配置组（修改/删除/切换并重启）。
- **编辑页 `/edit`**：新增或编辑配置组（支持“按组保存”与“保存并应用(重启)”）。内置**周边 Wi-Fi 扫描**功能。
- **资源页 `/resources`**：查看实时资源使用情况（Heap/PSRAM/Flash/运行时长等）。

### 步骤 4：配置组管理（新增/修改/删除/切换）
系统支持多配置组管理（最多 `8` 组），便于不同场景快速切换。

1. 在查看页点击 **“新增配置组”** 或 **“修改”** 进入编辑页。
2. 在编辑页填写参数：
   - **组名**：配置组显示名称。
   - **模式**：可选择 **AP 模式（仅热点）** 或 **中继模式（连接上游WiFi并共享热点）**。
   - **Wi-Fi 名称 (SSID)**：目标 Wi-Fi 名称（区分大小写）。
   - **Wi-Fi 密码**：目标 Wi-Fi 密码（开放网络可留空）。
   - **Wi-Fi 信道 (1-13)**：系统限制在 1~13 范围内。
   - **目标 BSSID (MAC 地址)**：格式如 `1A:2B:3C:4D:5E:6F`（可留空使用设备默认 BSSID）。
     - *高级用法*：支持多 MAC 地址轮换，用逗号分隔多个 MAC（例如 `1A:2B:3C:4D:5E:6F,1A:2B:3C:4D:5E:7F`），设备会在每次重启时自动轮换使用，增强隐蔽性。
3. 如果选择了**中继模式**，需要额外填写上游网络信息：
   - **上游 WiFi 名称 (SSID)**：要连接的互联网 Wi-Fi 名称。
   - **上游 WiFi 密码**：要连接的互联网 Wi-Fi 密码。
   - **上游 BSSID**：可选，锁定要连接的特定上游路由器的 MAC 地址。
   - *提示*：您可以点击 **“扫描附近 WiFi”** 按钮快速扫描周边网络，并点击“设为上游并填充”自动填入上述信息。
4. 保存方式说明：
   - **按组保存**：仅保存配置，不立即切换当前生效组。
   - **保存并应用(重启)**：保存后立即切换到该组并自动重启。
5. 在查看页还可执行：
   - **切换并重启**：切换到指定配置组并重启生效。
   - **删除**：删除非当前生效组（至少保留 1 组）。

### 步骤 5：资源监控与导出
1. 在查看页点击 **“资源监控”** 进入 `/resources`。
2. 可查看：
   - 运行时长、SDK 版本、芯片修订版本、当前连接设备数
   - Heap（可用/最小剩余/最大连续分配）
   - PSRAM（如果硬件支持）
   - 程序占用与 Flash 信息
3. 点击 **"导出信息(JSON)"** 可将资源快照 JSON 复制到剪贴板。

### 重新配置说明
如果后续需要再次修改参数：
1. 连接到设备当前正在广播的 Wi-Fi。
2. 浏览器访问 **`http://192.168.4.1`**。
3. 输入后台账号密码后即可进入管理页面继续调整。

## 6. 项目架构与技术特性

### 代码结构
```
Fake_WiFi/
├── Fake_WiFi.ino              # 主程序入口，WiFi初始化与路由配置
└── src/
    ├── common/
    │   └── AppState.h         # 全局状态定义（配置组结构体、模式常量等）
    ├── config/
    │   ├── ProfileStore.cpp   # 配置持久化（Preferences 存储/读取）
    │   └── ProfileStore.h
    └── web/
        ├── auth/
        │   ├── Auth.cpp       # HTTP Basic Auth 认证
        │   └── Auth.h
        ├── profiles/
        │   ├── ProfileHandlers.cpp  # 配置组管理（查看/编辑/保存/删除/切换）
        │   └── ProfileHandlers.h
        ├── resources/
        │   ├── ResourceHandlers.cpp # 资源监控页面与JSON导出
        │   └── ResourceHandlers.h
        ├── shared/
        │   ├── WebUiUtils.cpp # UI工具（基础样式、HTML转义等）
        │   └── WebUiUtils.h
        └── WebHandlers.h      # 路由声明
```

### 核心技术特性
* **双模式支持**：AP 模式（仅热点）与中继模式（AP+STA 双接口同时工作）
* **NAT 转发**：通过 `lwip_napt` 实现网络地址转换，为连接设备提供互联网访问（需开发环境支持）
* **BSSID 多 MAC 轮换**：支持配置多个 MAC 地址，设备自动轮换使用，增强隐蔽性
* **WiFi 扫描**：实时扫描周边网络，快速填充上游 WiFi 配置
* **配置组管理**：最多 8 组配置，支持快速切换与持久化存储
* **资源监控**：实时查看 Heap/PSRAM/Flash/运行时长等系统资源
* **低功耗优化**：默认降低 Wi-Fi 发射功率至 8.5dBm，减少峰值电流消耗
* **响应式 UI**：现代化 Web 界面，适配手机与桌面浏览器

## 7. 常见问题排查
* **找不到 COM 端口**：检查数据线是否具备数据传输功能，或者尝试重新运行 `install_ch340.ps1`。
* **页面打不开**：确保设备已连接到 ESP32 发出的 Wi-Fi，且没有自动切换回其他可上网 Wi-Fi。在中继模式下，ESP32 启动时连接上游 Wi-Fi 可能需要十余秒钟，请耐心等待。
* **弹出登录框但无法进入**：请确认输入的是 Basic Auth 账号密码（默认 `ad321/ad123`），或检查代码中是否已修改 `AUTH_USER` / `AUTH_PASS`。
* **看不到删除按钮**：当前生效组不能删除；请先切换到其他配置组后再删除该组。
* **信道设置后与输入不一致**：系统会将信道限制在 `1-13` 范围内。
* **打卡失败**：请确保 SSID、密码、信道、BSSID 与目标路由器一致；部分打卡应用会同时校验多项参数。
* **中继模式下无网络**：请确认您的开发环境支持且启用了 `lwip_napt`。若不支持，设备仅作为普通 AP 和 STA，连接设备将无法通过 ESP32 访问互联网。检查串口日志确认是否提示"NAT 启用任务已提交"。
* **BSSID 设置后每次重启都变化**：这是正常现象，设备会在您配置的多个 MAC 地址中自动轮换使用。如需固定 BSSID，请仅填写单个 MAC 地址。
* **信号较弱**：为降低峰值电流，默认发射功率设置为 8.5dBm。如需增强信号，可在 `Fake_WiFi.ino` 中修改 `WiFi.setTxPower()` 参数（如 `WIFI_POWER_11dBm`、`WIFI_POWER_15dBm` 等）。
* **设备频繁重启**：可能是电流不足导致，建议使用稳定电源供电，或降低发射功率。
